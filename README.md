# 🌅 Keith-Otter: Automated Morning Briefing System

An intelligent automation system that generates personalized daily briefings by aggregating RSS feeds, calendar events, and generating audio summaries.

## ✨ Features

- **📰 RSS Aggregation** - Fetch latest articles from multiple news sources
- **📅 Calendar Integration** - Include upcoming events in your briefing
- **🎙 Text-to-Speech** - Generate audio briefings (MP3)
- **🤖 GitHub Actions Automation** - Daily automatic generation at scheduled times
- **📊 Structured Output** - Multiple formats (Markdown, TXT, JSON)
- **🔄 Extensible** - Easy to add new data sources

## 🚀 Quick Start

### Prerequisites
- Python 3.8+
- pip/pipenv

### Installation

```bash
# Clone and setup
git clone https://github.com/fortunskitchenbar-ship-it/github.com-kmo1960.git
cd Keith-Otter

# Install dependencies
pip install -r requirements.txt

# Configure (optional)
cp config.example.json config.json
# Edit config.json with your RSS feeds
```

### Usage

```bash
# Generate today's briefing
python scripts/build_agenda.py

# Fetch RSS feeds only
python scripts/fetch_rss.py

# Fetch calendar events
python scripts/fetch_calendar.py

# Convert to speech
python scripts/text_to_speech.py --input agenda/briefing.txt --output agenda/briefing.mp3
```

## 📋 Configuration

Create a `config.json` file:

```json
{
  "rss_feeds": [
    "https://news.ycombinator.com/rss",
    "https://feeds.bloomberg.com/markets/news.rss"
  ],
  "output_dir": "agenda",
  "tts_enabled": true,
  "include_calendar": false
}
```

## 🔄 Automation

The GitHub Actions workflow (`daily_agenda.yml`) runs daily at **6:00 AM UTC**.

### Modify Schedule

Edit `.github/workflows/daily_agenda.yml`:

```yaml
schedule:
  - cron: '0 6 * * *'  # Change HH MM as needed
```

### Manual Trigger

```bash
gh workflow run daily_agenda.yml
```

## 📁 Output Files

Generated files in `agenda/` directory:

- `briefing.md` - Markdown formatted briefing
- `briefing.txt` - Plain text version
- `briefing.mp3` - Audio version (if TTS enabled)
- `rss_items.json` - Raw RSS feed data
- `calendar_events.json` - Calendar data
- `metadata.json` - Generation metadata

## 🔐 Secrets & Credentials

### For GitHub Actions:
- `GITHUB_TOKEN` - Auto-provided by GitHub
- `CALENDAR_TIMEZONE` - Optional timezone for calendar
- `RSS_FEEDS` - Optional RSS feed URLs

### For Local Calendar Integration:
1. Create Google Cloud Project
2. Enable Google Calendar API
3. Create OAuth credentials → save as `credentials.json`
4. First run prompts for authorization

## 🐛 Troubleshooting

### RSS Feed Issues
- Verify feed URLs are publicly accessible
- Check SSL certificates
- Increase timeout in `.env`: `RSS_TIMEOUT=20`

### Calendar API Errors
- Regenerate `credentials.json` from Google Cloud
- Ensure Calendar API is enabled
- Check OAuth scopes

### TTS Issues (Audio Generation)
- **Linux**: `sudo apt-get install espeak libespeak1`
- **macOS**: Uses native `say` command
- **Windows**: Uses SAPI (built-in)
